# 📘 DBMS Lab – Database Design Assignment (College Database)

## 🎯 Objective

The objective of this lab is to understand **Entity–Relationship (ER) modeling** and convert the ER diagram into **relational tables and SQL queries** for a college database system.

This assignment helps in designing a structured database and understanding relationships between different entities in a real-world college environment.

---

## 🧠 Concepts Covered

This experiment focuses on the following DBMS concepts:

* Entity–Relationship (ER) Model
* Entities and Attributes
* Primary Keys and Foreign Keys
* Relationships (1–1, 1–M, M–N)
* Cardinality
* Conversion of ER diagram to relational schema
* SQL table creation using constraints

---

## 🏫 College Database Description

The database represents a **college management system** consisting of multiple academic departments, students, faculty members, and courses.

### Main Entities in Database

* **Department** – Academic units like CSE, Mechanical, etc.
* **Student** – Students enrolled in different departments
* **Faculty** – Teaching staff of departments
* **Course** – Subjects offered by departments
* **Enrollment** – Relationship between students and courses

This system maintains structured information about academic operations in a college.

---

## 🧩 Lab Work Structure

This assignment is divided into **3 main parts**.

### 🔹 Part A: ER Diagram

In this part, an ER diagram is designed for the college database.

Tasks performed:

* Identify entities and attributes
* Define primary keys
* Show relationships between entities
* Define cardinality (1–1, 1–M, M–N)
* Create ER diagram using ER diagram tool (draw.io, etc.)

---

### 🔹 Part B: Relational Tables

In this section, the ER diagram is converted into relational schema.

Tasks performed:

* Convert each entity into table
* Define table attributes
* Assign primary keys
* Assign foreign keys
* Define relationships between tables

This step converts conceptual design into logical database structure.

---

### 🔹 Part C: SQL Implementation

In this section, relational tables are implemented using SQL.

Tasks performed:

* Writing CREATE TABLE queries
* Applying primary key constraints
* Applying foreign key constraints
* Choosing appropriate data types
* Maintaining referential integrity

All tables are created in Oracle SQL environment.
